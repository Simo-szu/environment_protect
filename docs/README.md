# 📚 项目文档和工具

这个文件夹包含项目的辅助文档和开发工具，**不会被推送到 GitHub**。

## 📁 文件夹结构

```
docs/
├── guides/          # 开发指南文档（如果有）
├── scripts/         # 辅助脚本
└── README.md        # 本文件
```

## 🛠️ 辅助脚本说明

### scripts/git-fix.sh
- **用途**：修复 Git 仓库问题
- **使用场景**：Git 索引损坏时使用
- **是否必需**：否，一次性工具

### scripts/fix-push.sh
- **用途**：诊断 GitHub 推送问题
- **使用场景**：无法推送代码时使用
- **是否必需**：否，问题已解决

### scripts/easy-push.sh
- **用途**：简化 Git 推送流程
- **使用场景**：快速提交和推送
- **是否必需**：否，可以用 GitHub Desktop 替代

### scripts/clean-macos-files.sh
- **用途**：清理 macOS 系统文件
- **使用场景**：定期清理 .DS_Store 和 ._ 文件
- **是否必需**：否，.gitignore 已配置自动忽略

## 🗑️ 可以删除吗？

**可以安全删除整个 docs 文件夹**，不会影响项目运行。

这些文件只是：
- 学习参考文档
- 一次性问题修复工具
- 可选的便利脚本

## 💡 为什么不推送到 GitHub？

1. **保持仓库整洁**：这些是本地辅助工具
2. **减少混乱**：团队成员不需要看到这些临时文件
3. **个性化工具**：每个人可能有不同的辅助工具

## 📖 重要文档位置

项目的核心文档在根目录：
- `README.md` - 项目说明
- `package.json` - 项目配置
- 其他中文文档 - 项目设计文档

## 🔧 如何使用脚本

如果需要使用某个脚本：

```bash
# 进入 docs/scripts 目录
cd docs/scripts

# 给脚本执行权限
chmod +x 脚本名.sh

# 运行脚本
./脚本名.sh
```

- 删除后不会影响 Git 历史
